# 0003. Repository + Unit of Work

- Статус: Accepted
- Дата: 2026-04-18

## Контекст

Ранее сервисы напрямую работали с `Session`, делали `commit` внутри мелких
функций (например, `add_request_log`), что ломало атомарность составных операций.

## Рассмотренные варианты

- Передавать `Session` во все сервисы, договориться о коммитах «наверху».
  Работает, но легко нарушить договорённость.
- **Repository + UoW**. Репозитории не коммитят; UoW агрегирует и держит границу транзакции.

## Решение

`SqlAlchemyUnitOfWork` как context manager. Внутри — репозитории
(`users`, `requests`, `request_logs`, `idempotency`, `outbox`). Сервис берёт UoW,
выполняет набор операций, вызывает `uow.commit()` ровно один раз.

## Последствия

- (+) Атомарность составных операций (заявка + audit + outbox в одной транзакции).
- (+) Тесты сервисов можно писать с fake-репозиториями (Protocol-интерфейсы).
- (−) Чуть больше boilerplate при добавлении новой сущности.
