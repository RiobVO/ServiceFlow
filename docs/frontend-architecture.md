# Frontend Architecture — ServiceFlow

## Цели
- Быстрое подключение к FastAPI через REST.
- Чёткое разделение ответственности (UI → страницы → сервисы).
- Готовность к масштабированию (новые модули, роли и workflows).

## Технологический стек
- React + TypeScript + Vite
- React Router для маршрутизации
- Fetch API для работы с REST

## Слои фронтенда
1. **Pages** — страницы приложения, держат сценарий и загрузку данных.
2. **Components** — переиспользуемые UI-блоки и layout.
3. **Services** — API-клиент и типы запросов/ответов.
4. **Utils** — общие хуки (local storage, контекст).

## Поток данных
```
UI (pages) → services/api.ts → FastAPI routers → services → DB
```

## Рекомендации
- Изолируйте бизнес-логику в сервисах, UI оставляйте декларативным.
- Храните API-ключ в localStorage, но для production добавьте SSO/refresh-токены.
- Используйте типы из `src/types` как единый источник правды для данных.
- Добавляйте новые доменные модули в виде страниц + сервисных функций.

## Расширение
- Добавьте Context/State manager (Zustand/Redux) при росте числа экранов.
- Введите server-side pagination для `/requests`.
- Добавьте toast-уведомления и скелетоны загрузки.
